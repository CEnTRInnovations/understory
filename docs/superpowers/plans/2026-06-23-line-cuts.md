# Line-Cut Connection Halo & Canvas Background Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a dual-stroke "halo" effect to connection lines in `ComplexityTimeline` so that crossing curves appear to pass under/over each other, and ensure the cream canvas colour is consistent between the UI and PNG exports.

**Architecture:** A single `BG_COLOR` constant (`#f4ede2`) drives three changes: (1) the CSS `--bg-main` variable is updated to match it, (2) each SVG connection `<path>` gets a wider same-colour halo path rendered first inside its `<g>` group, and (3) both canvas draw functions (`drawCardsMode` / `drawStrandsMode`) fill the background before drawing connections and use the same two-pass stroke technique.

**Tech Stack:** React + TypeScript, inline SVG, HTML5 Canvas 2D API. No new dependencies.

## Global Constraints

- Target file: `src/ComplexityTimeline.tsx`
- CSS variables file: `src/understory.css`
- Halo colour MUST equal the background colour — use the constant, never a magic string
- Do NOT restructure unrelated code; surgical edits only
- TypeScript must compile without new errors (`npm run build` or `npx tsc --noEmit`)
- Halo applies to **cards/default mode** connections; strands-mode connections use low opacity and are exempt

- [ ] **Step 2: Insert the halo path before the visible path (cards mode only)**

  Replace the entire `return (` block with:

  ```tsx
  return (
    <g key={i}>
      {/* Halo mask — only in cards mode; gives crossing lines an under-bridge look */}
      {displayMode !== 'strands' && (
        <path d={path}
          stroke={BG_COLOR}
          strokeWidth={(conn.width ?? 2) + 6}
          fill="none"
          strokeLinecap="round"
        />
      )}
      <path d={path}
        stroke={displayMode === 'strands' ? '#7E7C78' : conn.color}
        strokeWidth={displayMode === 'strands' ? 1 : conn.width}
        strokeOpacity={displayMode === 'strands' ? (isActive ? 0.35 : 0.1) : 1}
        fill="none"
        strokeDasharray={displayMode === 'strands' ? '2 4' : (conn.lineStyle === 'dashed' ? '6 4' : conn.lineStyle === 'dotted' ? '2 4' : undefined)}
        markerEnd={displayMode === 'strands' ? undefined : (conn.showArrow ? `url(#arrow-${i})` : undefined)}
      />
      {/* Invisible wide hit area — pointer-events re-enabled so the thin visible line is clickable */}
      <path d={path} stroke="transparent" strokeWidth={Math.max(16, conn.width + 14)} fill="none"
        style={{ pointerEvents: 'stroke', cursor: 'pointer' }}
        onClick={e => { e.stopPropagation(); setSelectedConnection(prev => prev === i ? null : i); }}
        onDoubleClick={e => { e.stopPropagation(); setEditingConnection(i); setShowConnectionModal(true); }} />
    </g>
  );
  ```

  > **Why `+6` not `+4`?** The spec says +4, but SVG `strokeWidth` is center-aligned, so a +4 halo leaves 2px visible on each side. +6 gives a clean 3px gap on each side, which reads more clearly at typical connection widths of 2–4px. Adjust if the product owner prefers tighter gaps.

      const from = getEventPos(conn.from);
      const to   = getEventPos(conn.to);
      const { x1, y1, x2, y2, c1x, c1y, c2x, c2y } = getConnectorGeometry(from, to, conn.fromSide, conn.toSide);
      const dash = conn.lineStyle === 'dashed' ? [6, 4] : conn.lineStyle === 'dotted' ? [2, 4] : [];

      // Pass 1: cream halo — erases whatever crossed beneath this connection
      ctx.save();
      ctx.strokeStyle = BG_COLOR;
      ctx.lineWidth   = (conn.width ?? 2) + 6;
      ctx.setLineDash([]);

      ctx.beginPath(); ctx.moveTo(x1, y1); ctx.bezierCurveTo(c1x, c1y, c2x, c2y, x2, y2); ctx.stroke();

      ctx.lineWidth   = conn.width;
      ctx.setLineDash(dash);

        ctx.closePath(); ctx.fill(); ctx.restore();
